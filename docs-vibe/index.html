🏗️ Kiến trúc 4 Tầng (4-Tier Runtime)

Kiến trúc phân tầng theo Microsoft Agent Mode. Giao tiếp qua contract, không gọi trực tiếp.

T1 — INTERFACE (Giao diện)

Validate input, classify prompt, format output. Cửa tiếp nhận mọi yêu cầu từ người dùng.

Agent: pxh-help
Input: Prompt tự nhiên / Lệnh / / @mention → Output: Request contract → T2
↓ Contract: Request
T2 — ORCHESTRATION (Điều phối)

Auto-route task, track state, enforce retry/recovery/reflection. Bộ não trung tâm.

Agent: pxh-pm (CEO)
Input: Request → Route → Policy check → Output: Task contract → T3
↓ Contract: Task
T3 — WORKERS (Nhân công)

Thực thi domain: code, test, review, build, UI/UX. 7 agents chuyên biệt.

Input: Task → Execute → Output: Result contract → T2
↕ Event / State
T4 — INFRASTRUCTURE (Hạ tầng)

Persist state, checkpoint, log, alerting. Ghi nhận mọi sự kiện, không quyết định.

Agent: pxh-save-history (Historian)
